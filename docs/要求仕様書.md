# 要求仕様書（v1.0）

## 1. アプリ概要

本アプリは「今日中心で、軽量に、タスクと予定を統合管理できる」スケジュール・メモアプリである。ユーザーが毎朝行う“今日やることの整理”を最小の負担で行えるよう設計する。

---

## 2. 目的（Purpose）

* タスク管理とスケジュール管理の分離による非効率を解消する。
* ユーザーが「今日やるべきこと」を直感的に把握できるようにする。
* 日常的に使い続けられる軽量なUIを提供する。
* 長期予定にも対応しつつ、日々の画面を重くしない構造を実現する。

---

## 3. スコープ（Scope）

### 本アプリが扱うもの

* 今日の予定
* 今日のタスク
* 短期（明日・今週・来週）の日付管理
* 長期予定の保持（深い階層で表示）

### 本アプリが扱わないもの（MVP）

* 高度なプロジェクト管理
* ガントチャート
* チーム連携
* 複雑な通知ロジック

---

## 4. ターゲットユーザー

### 学生・若者

* 日付番号を覚えない
* 今日・明日で動く
* 月カレンダーは重く感じる

### 社会人（特に情報系）

* スケジュール（Outlook）とタスク（メモ・OneNote）が分断
* 1日の優先順位が見えづらい
* 会議と差し込みタスクが多い

---

## 5. コンセプト

**「今日という単位で、生活を軽くする。」**

* 今日がアプリの中心
* 未来日を相対日付で扱う軽量UX
* 予定とタスクを生活の流れとして統合
* 月ビューは深い階層に隔離

---

## 6. 要求機能（機能要件）

### 6.1 今日（Daily Layer）

* 今日の予定一覧（時間表示）
* 今日のタスク一覧（チェック付き）
* 予定／タスクの折り畳み表示
* タスク3件表示を推奨（スクロール最小）
* 今日画面から予定・タスクの追加

### 6.2 短期（Week Layer）

* 週ストリップ（7日表示）
* 左右スワイプで「明日／今週／来週／再来週」へ移動
* 各日付に予定・タスクを配置可能

### 6.3 長期（Future Layer）

* 月カレンダー（深い階層でのみ表示）
* 日付ピッカー
* 長期予定の登録
* 重要イベントの通知（例：1週間前／1日前）

### 6.4 タスク管理

* タスク名
* メモ
* 相対期日（今日／明日／今週／来週）
* 完了チェック

### 6.5 予定管理

* タイトル
* 日付（絶対日付）
* 時間（任意）
* メモ（任意）

---

## 7. 非機能要件（NFR）

### 7.1 パフォーマンス

* 今日画面は 1 秒以内に表示されること

### 7.2 ユーザビリティ

* 1日1回以上開きたくなる操作性
* 1ステップでタスク追加が可能

### 7.3 継続性

* 情報が増えても今日画面が重くならない構造であること

---

## 8. 画面構成（概要）

1. **今日画面（メイン）**
2. **週ストリップ（短期）**
3. **長期予定（詳細カレンダー）**
4. **予定追加モーダル**
5. **タスク追加モーダル**

---

## 9. データ構造（概要）

```json
{
  "date": "2025-04-15",
  "tasks": [
    { "id": 1, "title": "買い物", "note": "牛乳", "due": "today", "done": false }
  ],
  "events": [
    { "id": 100, "title": "打ち合わせ", "time": "14:00", "memo": "Zoom" }
  ]
}
```
